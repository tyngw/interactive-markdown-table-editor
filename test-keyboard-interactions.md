# キーボードインタラクションと列幅自動調整テスト

このファイルは、改善されたキーボードインタラクションと列幅の自動調整機能をテストするためのものです。

## テーブル

| 短い | 中程度の長さのヘッダー | 非常に長いヘッダー名前でテストする場合 |
|------|----------------------|----------------------------------|
| A | データ1 | これは非常に長いテキストデータです |
| B | データ2 | 短いデータ |
| C | 中程度のデータ内容 | https://www.example.com/very/long/url |

## キーボードインタラクションのテスト手順

### Enter/Shift+Enterキーの動作テスト

1. VSCodeでこのファイルを開く
2. `Ctrl+Shift+P` → "Markdown Table Editor: Edit Table" を実行
3. 任意のセルを編集モードにする
4. **Enterキー**を押す → 編集確定し、下の行の同じ列に移動することを確認
5. **Shift+Enterキー**を押す → 改行が入力されることを確認
6. **Tabキー**を押す → 編集確定し、次のセルに移動し、前のセルの選択状態が解除されることを確認

### 列幅自動調整のテスト手順

1. 各列のヘッダー右端にマウスカーソルを合わせる
2. カーソルが列幅変更用（↔）になることを確認
3. **ダブルクリック**する → 列幅がコンテンツに合わせて自動調整されることを確認
4. 「短い」列 → 最小幅に近い値になる
5. 「中程度の長さのヘッダー」列 → ヘッダーテキストに合わせた幅になる
6. 「非常に長いヘッダー名前でテストする場合」列 → 適切に調整される（最大400px制限）

## 期待される動作

### キーボードインタラクション
- **Enter**: 編集確定 + 下の行に移動 + 編集開始
- **Shift+Enter**: 改行入力（編集継続）
- **Tab**: 編集確定 + 次のセル移動 + 編集開始 + 前セル選択解除
- **Escape**: 編集確定のみ

### 列幅自動調整
- リサイズハンドルでのダブルクリックでコンテンツに最適な幅に自動調整
- 最小幅80px、最大幅400pxの制限
- ヘッダーとデータセルの両方を考慮した幅計算
- 調整後は手動リサイズ可能状態になる
